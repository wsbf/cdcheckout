Deliverables

The four deliverables for this project are proposed as follows:

Phase 1
Here, core functionality is to be implemented and the existing database is to be extended.
An audit will take place to decide on action to take on the MySQL schema as it relates to storing CD checkout data, and then these modifications will be made.
Fundamental functionality such as authentication/authorization will be designed and implemented.
By the end of Phase 1, a command-line interface will be able to list albums categorized as "To Be Reviewed," authenticate users, and check out albums by albumID.
Phase 2
In this phase, unit tests will be performed to test Phase 1 functionality.
A GUI similar to that of the storyboard will be started. On the main view, the Library Checkout button and the search bar will be added, but will not work until Phase 3. The list of albums to be reviewed will be implemented, but will not include album art.
By the end of Phase 2, the program should be GUI-based, and allow users to check out an album by browsing the "To Be Reviewed" list, or by albumID.
Phase 3
In this phase, additional functionality will be added to the application to satisfy more requirements.
Searching will be implemented such that the album list filters as the user types on the keyboard.
Library checkout will be implemented (along with its associated view in the GUI).
If not already implemented, the ability to stay logged in to check out more albums will be added.
Appropriate test cases will be added and regression testing preformed.
Phase 4
In this phase, authentication will be extended with a card reader, if possible. The usefulness of all this depends on access to Clemson's database; as of now, we only have a partial list of CUIDs.
(aside) a non-functional requirement of this is that scanning a card should take less time than entering a username and password. Otherwise, it's not worth implementing, is it?
Test cases will be added and final testing done.
